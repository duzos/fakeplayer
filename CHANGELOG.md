# v2.0.7

- Upload local skin PNGs from disk via the skin select gui (#25). Click `UPLOAD` in the gallery, pick a 64x64 or 64x32 png up to 32 kb; the bytes go to the server, get persisted under `./fakeplayer/local-skins/`, and stream out to other clients on demand. Op-only by default; toggle `allowLocalSkinUploadOpOnly` in `config/players.json` to let everyone upload.
- Skin pipeline cleanup and perf pass (#40). Mints a unique resource location per registered skin and releases the prior one on re-register so re-skinned fake players no longer leak `DynamicTexture`s. Hashes urls and uploaded skin bytes as utf-8 / raw bytes so keys stay stable across locales. Skin downloads now use a single shared executor with shutdown, and the in-flight gate is an `AtomicBoolean`. Cached skin entries register synchronously on cache load instead of draining one-per-tick. Local-skin validation reads the png header directly so the upload path decodes the image once. Cached `SkinData` keys are normalised on construction and the skin select screen caches the key list, only rebuilding the preview entity when the selection changes. Backports the wide/slim renderer caching from later branches so each fake player no longer rebakes its model and layers every frame. Server now requires the uploader to be within 8 blocks of the target fake player on top of the existing op gate.
